## Features of Reminders

### Multiple reminder types
Lystaria supports different kinds of reminders depending on what you need. You can create normal reminders for general tasks, routine reminders for actions that need a checklist structure, and medication reminders if you are also using the Medication Manager. This allows the reminder system to support ordinary responsibilities as well as more specialized forms of care.

A normal reminder is exactly what it sounds like. It includes a title, description, optional checklist items, a date and time, and a chosen frequency. This makes it useful for everyday tasks that need structure without requiring extra complexity.

<img src="/img/screenshots/reminders/view.png" class="docs-image-single" />

A routine reminder is more structured. It requires checklist items, and in order to mark the reminder complete, all checklist items must be checked off first. This makes routine reminders especially useful for multi-step processes where full completion matters more than simply tapping one button.

<DotDivider />

### Flexible scheduling options
The Reminders system supports many different time types, which gives you a lot of freedom in how you build your schedule. You can create reminders that repeat every x days, every x weeks, every x months, or every x years. There is also an interval-based time type for reminders that need a repeating cadence outside of the more standard calendar patterns.

Monthly and yearly reminders also support additional customization, which makes them more useful for tasks that need a little more precision. This means the system is not limited to simple daily repeats, but can support schedules that reflect more realistic routines and longer-term responsibilities. The goal is to give you enough control that reminders feel adaptable rather than restrictive.

<DotDivider />

### Multiple fire times for one reminder
One reminder can support multiple fire times. This is especially useful for things that happen more than once in a day but still belong to the same responsibility. For example, if you take medicine twice daily, you could create one reminder called **Medicine** and set it to fire at 10 AM and again at 10 PM.

This helps reduce clutter because you do not need to create separate reminder objects for every occurrence. Instead, one reminder can represent one responsibility while still notifying you at more than one time. That makes the system cleaner and easier to maintain.

<DotDivider />

### Reminder card badges and statuses
Reminder cards include multiple badge types so you can understand what you are looking at quickly. Cards can show badges for reminder frequency, reminder type, and timing status. This gives each card more context without requiring you to open it first.

The timing status badges include **Due Now**, **Upcoming**, and **Overdue**. These badges help you immediately understand whether something needs attention right away, is coming soon, or has already passed its scheduled time. This makes scanning the reminder list much easier, especially when you are working with a lot of reminders at once.

<DotDivider />

### Connected reminders from other systems
Habits and events also support reminders, and those linked reminders will appear in the main reminder card list as well. This is important because it means the Reminders system is not isolated from the rest of the app. Instead, it acts more like a connected layer that helps bring different time-based responsibilities into one place.

Those linked reminders can also be completed from multiple views. You can complete them from the standard reminder cards list page, the Time Blocking page, or Kanban View. This gives you more flexibility in how you interact with your schedule and prevents the system from forcing you into one single workflow.

<DotDivider />

### Completion behavior
Whenever you complete a reminder, a gradient banner appears to confirm that it has been completed. This banner also shows the next run time and date, which helps you immediately understand what happens next. It is a small detail, but it makes the reminder system feel much clearer and more responsive.

The done button behaves differently depending on whether a reminder is one-time or recurring. If a reminder is recurring, the done button does not stay permanently filled, because that reminder will need to be completed again on its next occurrence. Only one-time reminders keep that completed state in a more permanent way, which makes sense because they do not need to return.

<DotDivider />

### Reschedule without breaking the original schedule
If you set a reminder for a specific day and time, and that day arrives but you need to move it without ruining the schedule you originally built, you can use the **Reschedule** button. This allows you to push the reminder back by date and time while still preserving the original schedule pattern you set for it.

This is especially helpful when life gets in the way and you need flexibility without having to rebuild the reminder itself. Instead of editing the schedule and accidentally changing the long-term pattern, you can simply move the current occurrence. That makes the system much more forgiving.

<DotDivider />

### Custom snoozing
If you only need to delay a reminder for a short amount of time, you can use the **Snooze** button. This lets you enter a custom snooze time so the reminder is pushed back temporarily. It is useful when you know you still want to do the task, but not at the exact moment it appears.

This keeps the reminder active while giving you breathing room. Rather than dismissing it or changing its actual structure, snoozing gives you a quick way to adjust the timing of the current moment. That makes it ideal for interruptions, delays, or times when you need a little flexibility.

<DotDivider />

### Creating and editing reminders
You can add reminders using the large floating plus button. This makes creating a new reminder quick and easy from the main reminders area. Once a reminder exists, you can edit it using the edit button on the reminder card.

This setup keeps the creation and editing flow straightforward. New reminders are always easy to start, while existing reminders remain easy to refine. That balance is important because reminders often need to evolve as routines change.

<DotDivider />

### Other reminder views and pages
The Reminders system includes header icon buttons that help you move to other related views and pages. These are important because reminders are not meant to live only in one flat list. Different views help you interact with the same reminder data in different ways depending on what kind of planning or visibility you need.
